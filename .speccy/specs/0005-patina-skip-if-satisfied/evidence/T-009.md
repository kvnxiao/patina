## Session 2026-06-03-T009-attempt-1

The "red" baseline is the pre-T-009 diff renderer
(`patina-cli/src/output/diff.rs`). `render` iterated `op.targets` and
called `render_operation` for **every** target unconditionally, with no
notion of per-target disposition and no summary line: a fully-satisfied
target still produced a `copy <target>` / content block, and the
Unchanged count was never reported.

T-009 makes `render` zip `op.targets` with the parallel `op.dispositions`
the T-004 classifier populates: a single-target `Unchanged` aggregate (or
an `Unchanged` tree leaf) is counted instead of rendered, and a single
`N unchanged entr{y,ies}.` line is emitted at the end when the count is
non-zero. `Create` / `Update` targets render exactly as before. For tree
modes the count is over materialized leaves (DEC-007): drifted leaves
render a block, clean leaves contribute to the count.
