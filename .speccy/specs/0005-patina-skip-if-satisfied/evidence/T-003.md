# Evidence for SPEC-0005 T-003

Paper trail for T-003: a plan-time classifier (`apply::classify::classify_leaf`)
that, given a resolved leaf `(mode, source, target, rendered-bytes?)`, reads
live target state and returns a `Disposition` (Create / Update / Unchanged),
reusing the exact `status::classify` primitives so "Unchanged" coincides with
status's "Clean".

T-003 is a pure-function library task with no CLI surface: the classifier is
not yet wired into the plan path (that is T-004). The proof is therefore the
unit-test matrix — each mode × {Create, Update, Unchanged} plus the
shared-primitive tie — exercised under the standard `cargo test` hygiene
suite, not a CLI red/green run. CHK-001 / CHK-002 / CHK-003 under REQ-001 are
labelled `hygiene` in the journal roll call and are backed by the unit tests
captured below.

## Session 2026-06-02-T003-attempt-1

### Shared-primitive seam (the REQ-001 "no third definition of matches" tie)

The symlink and content comparisons were factored out of `status::classify`
into two `pub(crate)` seams both call sites share:

- `status::classify::symlink_matches(target, desired)` — the `simplified_str`
  link-target compare (was inline in `classify`'s `Symlink` arm).
- `status::classify::content_matches(target, &hash)` — the `blake3`
  content-hash compare (was inline in `classify`'s `Content` arm).

`status::classify::classify` now calls both, so the status path is unchanged
behaviourally; `apply::classify::classify_leaf` calls the same two, so
"Unchanged" is the same decision as "Clean" with no copied compare.

Matrix coverage against the REQ-001 `<done-when>` bullets:

- **CHK-001 (symlink Unchanged / Update / Create):** `symlink_*` tests —
  a link pointing at the source classifies Unchanged; pointing elsewhere or
  replaced by a regular file classifies Update; an absent target classifies
  Create.
- **CHK-002 (copy/template Unchanged / Update / Create):** `copy_*` and
  `template_*` tests — hash-equal bytes (copy) and byte-equal rendered output
  (template) classify Unchanged; differing bytes classify Update; absent
  classifies Create.
- **CHK-003 (per-leaf tree classification):** `copy_tree_leaf_classifies_like_copy`
  — a copy-tree leaf is a single regular-file comparison, so a matching leaf
  classifies Unchanged and a drifted leaf classifies Update independently,
  which is what the engine (T-004) drives per leaf to land "one drifted leaf
  Update, the other two Unchanged".

### The Clean ⇔ Unchanged tie (REQ-001 shared-primitive bullet)

`state_status_calls_clean_classifies_unchanged_symlink` and
`...content` drive **both** `status::classify::classify` (returns `Clean`) and
`apply::classify::classify_leaf` (returns `Unchanged`) on the exact same live
filesystem state and assert they agree. This is the test that fails if a
future edit reintroduces a second definition of "matches".

## Hygiene-gate evidence

The project's canonical gate is `just check` (= `just lint` + `just test`),
which is stricter than the four raw cargo commands: `just lint` runs nightly
`fmt --all --check`, clippy (cross-compiled to `x86_64-unknown-linux-gnu` and
`x86_64-pc-windows-gnu` so the `#[cfg(windows)]` symlink-creation test helper
is linted on both targets), `RUSTDOCFLAGS=-D warnings cargo doc`, and
`cargo deny check`; `just test` is `cargo test --workspace --locked`.

```
$ just check
cargo +nightly fmt --all --check          # exit 0
cargo clippy ... --target x86_64-unknown-linux-gnu -- -D warnings   # exit 0
cargo clippy ... --target x86_64-pc-windows-gnu  -- -D warnings     # exit 0
RUSTDOCFLAGS=-D warnings cargo doc --workspace --no-deps --all-features  # exit 0
cargo deny check                          # exit 0
cargo test --workspace --locked           # all passed
$ echo $?
0
```

All gates exited 0 after the T-003 diff landed. No new dependencies were
added (`blake3` via `content_hash`, `fs-err`, and `camino` are all
pre-existing).
