# Evidence for SPEC-0003 T-007

Red-then-green paper trail for the watcher's subscription-set
computation (REQ-005). This task lands the pure mapping from a committed
journal `ApplyRecord` to the ordered, de-duplicated set of paths the
watcher subscribes to: every target's canonical source, every
content (non-symlink) target path, and the `<state>/patina/journal/`
directory — and it omits symlink target paths (DEC-008). It does no
`notify` wiring; T-008 hands the set to the debouncer and T-009 does the
journal-rescan-on-new-COMMIT that CHK-017 inspects.

The two `<task-scenarios>` in TASKS.md are the contract; both map onto
CHK-009's path-set arithmetic.

### Supporting unit tests

Three further lib unit tests pin adjacent properties the two scenarios
imply: a content target contributes both its source and its target path
(`content_target_subscribes_both_source_and_target`); two entries sharing
one source collapse to a single first-occurrence subscription
(`duplicate_source_paths_collapse`, exercising the de-dup invariant the
scenarios' "exactly five / two" counts depend on); and an empty record
still yields the journal-rescan subscription
(`empty_record_yields_only_journal_dir`).

```

```

### CHK-009 / CHK-017 status

CHK-009's *full* surface — a running foreground watcher whose subscription
list is read back from the `tracing` log — requires the watcher event loop
and stderr layer that land in T-008. T-007 emits the set as a
`watch_subscriptions` info event (`target: "patina_core"`, fields `count`
and tab-joined `paths`) so T-008's harness can inspect it, and proves the
*path-set arithmetic* CHK-009 asserts (exactly four watched paths plus the
journal dir = five) at the unit level. CHK-017 (journal-rescan on a new
`.COMMIT`) is wholly T-009's: it needs the live watcher re-reading the
latest commit. Both are carried as deferred here.
