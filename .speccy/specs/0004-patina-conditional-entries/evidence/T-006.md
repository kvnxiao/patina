# Evidence for SPEC-0004 T-006

Paper trail for T-006: the plan-time source existence-and-kind check, step
(3) of the REQ-009 order, run after a surviving (`when`-true or no-`when`)
entry is canonicalized (REQ-002 / REQ-009 / DEC-008). T-005 inserted the
`when` gate so a `when`-false entry returns `Ok(None)` and is never
canonicalized; T-006 adds the validation in the `Some` branch ‚Äî so a
gated-off entry is never kind-checked (CHK-022), and a surviving entry whose
source is wrong-shaped or absent fails during planning, before the advisory
lock, the journal flush, or any mutation.

## Production change

`patina-core/src/error.rs`:

- New `EngineError::SourceKindMismatch { path, found, declared_table,
  expected_table }` ‚Äî a `[[file]]` source that is a directory, or a
  `[[directory]]` source that is a file. The `Display` names the source
  `path`, the on-disk `found` kind, and both the `declared_table` and the
  `expected_table` so the message directs the author to the correct table.
- New `EngineError::SourceNotFound { path }` ‚Äî a surviving entry whose
  canonical source does not exist on disk. (Field is `path`, not `source`:
  `thiserror` treats a `source` field as a `#[source]` error wrapper, and
  `Utf8PathBuf` is not an `Error`.)

`patina-core/src/apply/engine.rs`:

- `resolve_entry` now calls `validate_source_kind(&source, entry.kind)?`
  immediately after `canonicalize(&module_path.join(&entry.source))` and
  before the targets are expanded ‚Äî step (3) of REQ-009, on the
  already-resolved source (no extra IO pass beyond the one `stat`).
- `validate_source_kind(source, kind)` probes the canonical source with
  `fs_err::metadata` (which follows symlinks, so the kind validated is the
  kind the executor will materialize). `NotFound` ‚Üí `SourceNotFound`; any
  other IO error ‚Üí `EngineError::Path(PathError::Filesystem { .. })`; a
  `[[file]]` whose metadata `is_dir()` ‚Üí `SourceKindMismatch` directing to
  `[[directory]]`; a `[[directory]]` whose metadata is not a dir ‚Üí
  `SourceKindMismatch` directing to `[[file]]`. The explicit existence probe
  is necessary because `paths::canonicalize` falls back to *lexical*
  resolution for a non-existent path, so a missing source does not fail at
  canonicalization.

Because the check lives inside `resolve_entry`, which `gate_and_resolve_entry`
calls only on the `when`-survivor (`Some`) branch, a `when`-false entry is
never canonicalized or validated (REQ-009 ordering / CHK-022). The executor's
materialize-time existence check (`apply/symlink.rs` `ExecutorError::Source
Missing`, and the copy/template equivalents) is left untouched as the
materialize-time TOCTOU backstop (DEC-008).

## Session 2026-06-01-T006-attempt-1

### Unit test ‚Äî `[[file]]` directory source is a kind mismatch

`patina-core/src/apply/engine.rs::tests::file_entry_with_directory_source_is_a_kind_mismatch`
creates a directory `confdir` and calls `validate_source_kind(&confdir,
EntryKind::File)`, asserting the error is `SourceKindMismatch` whose `path`
is the source, `found == "directory"`, `declared_table == "[[file]]"`, and
`expected_table == "[[directory]]"`.

Mutation check: flipping the `is_dir()` condition (or dropping the `File`
arm) yields `Ok(())` and fails the `expect_err`.

### Unit test ‚Äî `[[directory]]` file source is a kind mismatch (symmetric)

`patina-core/src/apply/engine.rs::tests::directory_entry_with_file_source_is_a_kind_mismatch`
writes a regular file `gitconfig` and calls `validate_source_kind(&gitconfig,
EntryKind::Directory)`, asserting `SourceKindMismatch` with `found == "file"`,
`declared_table == "[[directory]]"`, `expected_table == "[[file]]"`.

Mutation check: a `!metadata.is_dir()` typo'd to `metadata.is_dir()` yields
`Ok(())` and fails the `expect_err`.

### Unit test ‚Äî absent source is `SourceNotFound`, not a kind mismatch

`patina-core/src/apply/engine.rs::tests::absent_source_is_source_not_found`
calls `validate_source_kind(&ghost, EntryKind::File)` on a path that was never
created, asserting `SourceNotFound { path }` naming the source.

Mutation check: returning the kind-mismatch variant for the not-found branch,
or relying on canonicalization to fail (it does not ‚Äî it resolves lexically),
fails the variant match.

### Unit test ‚Äî matching kinds validate cleanly

`patina-core/src/apply/engine.rs::tests::matching_kinds_validate_cleanly`
asserts a real file under `EntryKind::File` and a real directory under
`EntryKind::Directory` both return `Ok(())`. This is the control that guards
against the check rejecting a valid entry.

Mutation check: an over-eager check that rejected any source would fail both
`expect`s.

### CHK-004 ‚Äî `[[file]]` directory source fails, names source and `[[directory]]`, no journal artifact

`patina-cli/tests/source_kind_validation.rs::file_entry_with_directory_source_fails_and_directs_to_directory_table`
drives the real `PATINA_REPO=<tempdir> patina apply --yes` over a fixture
whose `[[file]]` entry has `source = "confdir"` where `confdir` is a
directory. Asserts: exit 1; stderr contains `confdir` and `[[directory]]`;
the target is not created; and the journal dir holds no `*.plan` / `*.COMMIT`
for the run (`assert_no_journal_artifacts`).

Mutation check: without the plan-time check the apply reaches the executor,
which fails later ‚Äî but only after the journal plan is flushed, so the
no-journal-artifact assertion fails red; and the error message would not name
`[[directory]]`.

### CHK-005 ‚Äî `[[directory]]` file source directs the author to `[[file]]`

`patina-cli/tests/source_kind_validation.rs::directory_entry_with_file_source_fails_and_directs_to_file_table`
drives `patina apply --yes` over a `[[directory]]` entry whose `source =
"gitconfig"` is a regular file. Asserts exit 1, stderr contains `[[file]]`,
and no journal artifact.

Mutation check: dropping the `Directory` arm of `validate_source_kind` lets
the directory executor run against a file source and the `[[file]]` substring
never appears.

### CHK-018 ‚Äî `when`-true entry with absent source is `SourceNotFound`, no journal artifact

`patina-cli/tests/source_kind_validation.rs::when_true_entry_with_absent_source_fails_as_source_not_found`
drives `patina apply --yes` over a `[[file]]` with `source = "ghost"` and no
`when`, where `ghost` does not exist. Asserts exit 1, stderr names `ghost`,
the target is not created, and no journal artifact.

Mutation check: relying on the executor's materialize-time check alone would
flush the plan journal first, failing the no-artifact assertion.

### CHK-022 ‚Äî `when`-false wrong-OS absent-source `[[directory]]` exits 0 with no kind/missing-source error

`patina-cli/tests/source_kind_validation.rs::when_false_entry_with_absent_wrong_kind_source_is_not_validated`
drives `patina apply --yes` over a `[[directory]]` whose
`when = "patina.os == 'definitely-not-this-os'"` and whose `source` does not
exist. Asserts exit 0, stderr contains neither `does not exist` nor
`[[file]]`, and no target is created. This is the load-bearing ordering
proof: the kind/existence check (T-006) must run *after* the T-005 `when`
gate, so a gated-off entry is never reached.

Mutation check: moving `validate_source_kind` above the `when` gate (e.g.
into `gate_and_resolve_entry` before `eval_when`) makes this exit 1 with a
`SourceNotFound` error ‚Äî failing both the exit-0 and the no-error assertions.
