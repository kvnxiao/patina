# Evidence for SPEC-0004 T-007

Paper trail for T-007: the `symlink-tree` per-leaf directory executor
(REQ-006 / DEC-005 / DEC-007). A `[[directory]]` entry with
`mode = "symlink-tree"` walks its source directory and creates one symbolic
link per leaf file at the mirrored target path, leaving intermediate target
directories real. Empty source subdirectories are skipped; a pre-existing
regular file at a leaf target is backed up by the engine and replaced by the
link; a re-apply over unchanged source is a no-op.

## Production change

`patina-core/src/apply/symlink.rs`:

- New `pub(super) fn tree_symlink(source, targets)` — the per-leaf executor.
  Rejects a non-directory source with `ExecutorError::NotADirectory` and a
  missing source with `ExecutorError::SourceMissing` (same `symlink_metadata`
  classification `dir_symlink` already uses). Collects the source leaves once
  via the shared `super::walk_files` (deterministic sorted order, regular
  files only), then for each target links each leaf through the existing
  `link_file`. Reuses `link_file` and `walk_files` verbatim — the same
  walk-and-link path the default-mode `per_file_symlink` directory branch
  already used pre-split.
- Module doc updated to list the third symlink executor (`SymlinkTree`).

`patina-core/src/apply/mod.rs`:

- `materialize` now dispatches `FileMode::SymlinkTree => symlink::tree_symlink(..)`,
  replacing the `ExecutorError::SymlinkTreeNotImplemented` guard left by
  T-002.
- The now-unreferenced `ExecutorError::SymlinkTreeNotImplemented` variant is
  removed.
- Module doc updated from "five modes" to "six modes" and the `symlink`
  submodule line now names `SymlinkTree`.

The `engine.rs` plan-time dispatch needed no change: T-002 already maps
`FileMode::SymlinkTree` to `PlannedOperation::symlink` (the correct durable
op shape — a `symlink-tree` materializes symlinks), and the per-leaf
granularity (one `CompletionRecord` per leaf) flows from `link_file`
unchanged into the commit record and progress cursor.

## Walk semantics confirming the empty-subdir skip

`walk_files` (`apply/mod.rs`) descends directories and pushes only regular
files into the relative-path list. An empty source subdirectory therefore
yields no entry, so `tree_symlink` creates neither a target directory nor a
link for it — REQ-006's "empty subdirectories skipped" falls out of the
existing walk with no special-casing. `link_file`'s `ensure_parent` creates
intermediate target directories on demand as real directories.

## Tests

### Executor unit tests (`patina-core/src/apply/symlink.rs`, `#[cfg(test)]`)

- `tree_symlink_links_each_leaf_and_keeps_intermediate_dirs_real` — source
  `a.conf` + `sub/b.conf` → two records, both leaves are symlinks resolving
  (canonicalized) to their source files, and both `dest` and `dest/sub` are
  real directories (`is_dir() && !is_symlink()`). CHK-012 executor leg.
- `tree_symlink_skips_empty_source_subdirectories` — source with an empty
  `empty/` dir and one real leaf → one record, `dest/empty` does not exist.
- `tree_symlink_replaces_pre_existing_regular_file_leaf` — a pre-existing
  regular file at `dest/a.conf` is replaced by a symlink to the source (the
  engine has already backed it up; `link_file` clears the path). CHK-013
  executor leg.
- `tree_symlink_rejects_file_source` — a regular-file source →
  `ExecutorError::NotADirectory`.
- `tree_symlink_missing_source_is_typed` — an absent source →
  `ExecutorError::SourceMissing`.
- `tree_symlink_re_apply_over_unchanged_source_is_a_noop` — a second
  `tree_symlink` over the same source and target succeeds and leaves both
  leaf links resolving to the source (idempotency).

### CLI integration tests (`patina-cli/tests/symlink_tree.rs`)

Each drives `PATINA_REPO=<tempdir> patina apply --yes` as a subprocess (stdin
not a TTY, exercising the non-interactive path) over a fixture repo with a
`symlink-tree` `[[directory]]` entry.

- `symlink_tree_links_each_leaf_and_keeps_intermediate_dirs_real` — CHK-012:
  `~/d/a.conf` and `~/d/sub/b.conf` are symlinks resolving (canonicalized) to
  the source leaves; `~/d` and `~/d/sub` are real directories.
- `symlink_tree_backs_up_pre_existing_leaf_and_replaces_it_with_a_link` —
  CHK-013: a pre-existing regular file at `~/d/a.conf` is afterward a symlink
  to the source; the prior bytes (`original`) are recovered by a subsequent
  `patina rollback --yes` restoring the leaf to a regular file with those
  bytes — exercising the real backup machinery rather than poking at the
  binary backup tree internals.
- `symlink_tree_skips_empty_source_subdirectory` — an empty `empty/` source
  subdir produces no `~/d/empty` target directory; the real leaf still links.
- `symlink_tree_re_apply_over_unchanged_source_is_a_noop` — two consecutive
  `patina apply --yes` runs both exit 0 and the leaf links resolve to the
  source.
