# Evidence for SPEC-0002 T-010

Paper trail for T-010: the read-only `patina doctor` path (REQ-005,
REQ-010) — a `Doctor` subcommand + `DoctorArgs` (`--fix`/`--json`/`--yes`)
in `cli.rs`, a new `patina-cli/src/cmd/doctor.rs`, and dispatch in
`main.rs`. The `--fix` remediation is out of scope (T-011); this task wires
only the read-only finding emission, the shared-lock acquisition (REQ-009),
and the deterministic `--json` document (REQ-010).

## Design: pure finding computation over gathered inputs

The finding set is decided by `compute_findings(&Inputs) -> Vec<Finding>`,
which is pure over its argument — no filesystem, registry, or environment
access. `run` gathers the host-state `Inputs` (platform, Developer Mode
status, OS-build support, resolved repo path, whether the repo declares a
symlink entry, whether the `default_repo` pointer exists), then calls the
pure function. This makes the whole v1.0 finding set — including the three
`DOC-WIN-*` findings — unit-testable on the macOS/Linux CI, with the
Windows-specific reads supplied as plain `Inputs` fields rather than real
registry probes.

## Coverage note: CHK-010 on a non-Windows CI

CHK-010 is a full run on a Windows host with Developer Mode OFF and a
repository declaring a `mode = "symlink"` entry. The macOS/Linux CI has no
Windows host and no Developer Mode registry, so the end-to-end integration
form ships as `#[cfg(windows)]` `#[ignore]`
(`windows_devmode_off_with_symlink_repo_warns` in
`patina-cli/tests/doctor_cli.rs`). The finding *logic* CHK-010 asserts —
symlink-declared + Developer-Mode-disabled ⇒ `DOC-WIN-DEVMODE` warning whose
message names `Developer Mode` and the registry path — IS demonstrated on
the CI host by the pure unit test
`windows_devmode_finding_requires_symlink_and_disabled` (it fills
`Inputs { is_windows: true, dev_mode: Disabled, repo_declares_symlink: true,
.. }` and asserts the warning's code/level/message).

## Red/green note

- CHK-018 determinism (`doctor_json_is_byte_identical_across_runs`): would go
  red if any non-deterministic field (timestamp, PID, repo-path ordering)
  entered the `--json` document; green because the only finding for a
