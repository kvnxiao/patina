# Evidence for SPEC-0002 T-003

Red-then-green paper trail for `patina init` and the generalized
integration harness (REQ-001, REQ-010). The red phase runs the new
`patina-cli/tests/init_cli.rs` crate against the pre-change binary
(the `Init` subcommand absent); the green phase runs it against the
implemented command. The four tests back the `demonstrated` CHK
labels in `journal/T-003.md`.

## Red capture method

The implementation source (`cmd/init.rs`, and the `Init` arm in
`cli.rs` / `cmd/mod.rs` / `main.rs`) was removed (untracked
`init.rs` moved aside, tracked edits `git stash`ed) while keeping
the new `tests/init_cli.rs` crate present, then the crate was run.
Every test fails at its exit-code assertion: the pre-change `patina`
has no `init` subcommand, so clap rejects the argument and the
process exits `2`, not the `0`/`1` the tests expect.
