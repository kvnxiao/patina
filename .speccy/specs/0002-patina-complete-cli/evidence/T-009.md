# Evidence for SPEC-0002 T-009

Paper trail for T-009: wiring the Developer Mode elevation flow (REQ-007)
end-to-end — the `ShellExecuteEx`/`runas` helper launch in
`patina-core::windows::elevate`, the Windows-only engine gate in
`apply/engine.rs::execute`, and the CLI orchestration in
`patina-cli/src/cmd/apply.rs`. This task connects T-007's detection
(`decide_symlink_gate`, `HostDevModeProbe`) to T-008's standalone
`patina-elevate.exe` helper.

## Coverage note

REQ-007's CHK-012 and CHK-013 are full end-to-end runs on a Windows host
with Developer Mode off/on and a real UAC consent dialog. The macOS/Linux
CI has no Windows host and cannot raise a UAC prompt, so both scenarios
ship as `#[cfg(windows)]` `#[ignore]` host tests
(`windows_declined_uac_exits_5_and_creates_no_symlink`,
`windows_dev_mode_on_creates_symlink_without_prompt` in
`patina-cli/tests/apply_cli.rs`) — `judgment-only` for the CI host, with
the `#[cfg(windows)]` code type-checked + clippy-clean via cross-compile
(see below). The cross-platform guarantee (task scenario 1) IS
demonstrated on the CI host.

## Scenario roll call

### Task scenario 1 — non-Windows skips the dev-mode flow (demonstrated)

`non_windows_symlink_apply_skips_dev_mode_flow` in
`patina-cli/tests/apply_cli.rs` (`#[cfg(not(windows))]`): a symlink
`[[file]]` apply with `--yes` on the CI host exits 0, the target is a real
symbolic link (proving the gate proceeded rather than refusing to mutate),
and neither `patina-elevate` nor `Developer Mode` appears in combined
stdout/stderr. This is positive proof that `decide_symlink_gate` returns
`Proceed` on a `NotWindows` probe with no registry read and no helper
spawn.

### CHK-012 — declined UAC → no symlink, exit 5, stderr substrings (judgment-only on CI)

`windows_declined_uac_exits_5_and_creates_no_symlink`
(`#[cfg(windows)]` `#[ignore]`): asserts exit 5, no symlink at the target,
and that stderr contains both `Developer Mode` and `patina doctor --fix`.
The CLI maps the helper's `ElevationOutcome::Declined` to
`ExitCode::UserDeclined` (5) and emits the substring-bearing warning in
`drive_elevation`. Requires a Windows host + declined UAC dialog.

### CHK-013 — Developer Mode ON → symlink created, no prompt/helper (judgment-only on CI)

`windows_dev_mode_on_creates_symlink_without_prompt`
(`#[cfg(windows)]` `#[ignore]`): asserts exit 0, a real symlink at the
target, and no `patina-elevate` mention. With Developer Mode on,
`decide_symlink_gate` returns `Proceed`, so the CLI never calls
`launch_elevate_helper`. Requires a Windows host with Developer Mode
enabled.
