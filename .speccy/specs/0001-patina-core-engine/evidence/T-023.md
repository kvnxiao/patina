### Scenario 1 — The four print macros are denied; injecting one fails clippy naming the file (CHK-053 enforcement half, CHK-054)

<red>
Pre-edit: `clippy.toml` carried only `allow-expect-in-tests = true`
with no `disallowed-macros` list. A raw `println!` anywhere in the
workspace passed clippy clean. Confirmed by adding the list and then
running the workspace clippy against the `lock_helper` example, which
already contained three real `println!` / `eprintln!` calls — they had
been silently permitted at HEAD:

```
$ cargo clippy --workspace --all-targets --locked -- -D warnings
error: use of a disallowed macro `std::println`
   --> patina-core\examples\lock_helper.rs:149:5
error: use of a disallowed macro `std::println`
   --> patina-core\examples\lock_helper.rs:159:5
error: use of a disallowed macro `std::eprintln`
   --> patina-core\examples\lock_helper.rs:167:9
error: could not compile `patina-core` (example "lock_helper") due to 3 previous errors

This is the red baseline: before the list existed those three sites
were invisible to clippy; the moment the list is added they fail. It
proves the list bites and the diagnostic names the offending file +
line.
</red>

`lock_helper.rs` carry scoped
`#[expect(clippy::disallowed_macros, reason = "...")]` carve-outs and

```

### Scenario 2 — `tracing`-style macros and the scoped carve-out stay clean (REQ-026 behavior sibling scenarios)

<red>
Without distinguishing the disallowed set, a naive ban could either
(a) miss the carve-out shape (leaving a real `println!` site
unsuppressable without a bare `#[allow]`, which the workspace's
`allow_attributes` deny-lint itself rejects) or (b) over-fire on
`tracing` macros. The risk is concrete because the project denies
`allow_attributes` / `allow_attributes_without_reason`, so the only
legal suppression is `#[expect(..., reason = "...")]` — which itself
fails `-D warnings` if the expectation is unfulfilled.
</red>

<green>
The integration test
`tracing_macro_and_scoped_expect_stay_clean` plants a non-listed macro
(`info!`, a local `macro_rules` stub standing in for a `tracing`
macro) and a genuine `println!` guarded by a module-scoped
`#[expect(clippy::disallowed_macros, reason = "...")]`, then asserts
clippy exits 0 with zero `disallowed_macros` spans. This proves the
non-listed macro never fires the lint and the scoped expect both
suppresses the lint and is fulfilled (no
`unfulfilled_lint_expectations` warning).

The same carve-out shape is exercised against the real tree by the
three `#[expect(clippy::disallowed_macros, reason = "...")]` blocks in
`patina-core/examples/lock_helper.rs`, which the green workspace
clippy run above accepts.
</green>
