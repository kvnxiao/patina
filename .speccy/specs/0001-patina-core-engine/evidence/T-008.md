# Evidence — SPEC-0001 / T-008

Paper trail for the single strict-undefined `MiniJinja` environment
(`patina-core::template`) covering REQ-009. T-008 owns one shared
`minijinja::Environment` that renders `*.tmpl` bodies and evaluates
`[[file]]` / `[[hook]]` `when` predicates against the six-layer
variable resolver from T-006.

CHK-019 names `patina apply --yes` exiting 1 with `user_email` on
stderr. The apply pipeline (file-mode executors T-014, hook execution
T-015) and the CLI surface (T-016) are downstream — there is no apply
plan computation to invoke yet. This task ships the plan-computation
contract those layers depend on: an undefined reference in a template
body produces a typed `TemplateError::UndefinedVariable` whose Display
names the offending variable. The CLI maps that error to exit 1 with
the name on stderr once T-016 lands. CHK-020 is fully demonstrated at
the engine layer here.

## Behaviour notes

- The engine uses `UndefinedBehavior::SemiStrict`, not `Strict`.
  `Strict` errors the moment an undefined value is *tested* — including
  the `{% if defined %}` condition — which breaks the REQ-009
  `{% else %}` carve-out. `SemiStrict` errors on emit/coercion of an
  undefined value (CHK-019) while treating a bare `{% if missing %}`
  test as falsy (the carve-out). See the module docs in
  `patina-core/src/template/mod.rs` for the full rationale.
- `compile_expression().eval()` returns an *undefined value* (not an
  error) when a `when` expression resolves to undefined — e.g. the
  short-circuit result of `<true> and missing_var`. `Engine::eval_when`
  converts that undefined result into the same
  `TemplateError::UndefinedVariable` so a `when` touching an undefined
  variable is reported, not silently treated as false (CHK-020).
- `MiniJinja`'s `UndefinedError` carries only the generic text
  `"undefined value"` and does not name the missing variable. The
  template context is a dynamic `minijinja::Object` that records every
  key resolving to undefined in a per-evaluation tracker; after a
  failure the recorded names are read back to build the typed error.
