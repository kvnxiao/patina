### Dep-tree scenarios (task-level, not a numbered CHK)

- `patina-core/Cargo.toml` declares `thiserror` and `tokio` as direct
  deps; **no** `anyhow` entry — `anyhow` is binary-only per the
  project's hard rule.
- `patina-cli/Cargo.toml` declares `anyhow`, `tokio`, and a path
  dependency on `patina-core`.
- `tokio` in `patina-core` enables exactly the feature set named by the
  task body: `rt-multi-thread`, `fs`, `process`, `signal`, `sync`,
  `time`, `io-util`, `macros`.

## REQ-002 — Async tokio foundation

### CHK-003 — `cargo check --workspace --locked` exits 0; lib declares three `pub async fn`

```
$ cargo check --workspace --locked
    Finished ...
```

`patina-core/src/lib.rs` declares:

- `pub async fn apply(_options: ApplyOptions) -> Result<(), EngineError>`
- `pub async fn status(_options: StatusOptions) -> Result<(), EngineError>`
- `pub async fn rollback(_options: RollbackOptions) -> Result<(), EngineError>`

Each returns a typed `Result<_, EngineError>` (see
`patina-core/src/error.rs`). The placeholder body returns
`EngineError::NotImplemented("...")` rather than `todo!()` / `panic!()`,
honouring REQ-024's no-panic rule even though T-002 is the task that
flips `todo` / `unimplemented` from `warn` to `deny`.

### CHK-004 — `patina-cli/src/main.rs` is `#[tokio::main]` and awaits `patina-core`

```rust
#[tokio::main]
async fn main() -> Result<()> {
    ...
    let result = patina_core::status(patina_core::StatusOptions::default()).await;
    result.context("patina-core::status failed")?;
    Ok(())
}
```

The binary awaits `patina_core::status` — wiring proof, not real
behaviour. The `EngineError::NotImplemented` placeholder surfaces
through `anyhow::Context` and exits non-zero at runtime; future tasks
replace it.
