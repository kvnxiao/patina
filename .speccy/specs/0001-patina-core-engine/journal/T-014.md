  mode. Symlink-family modes route through `create_symlink`, which selects
  the OS primitive (`fs_err::os::unix::fs::symlink` on Unix;
  `symlink_file`/`symlink_dir` on Windows per a `LinkKind`) and maps a
  Windows `PermissionDenied` to `ExecutorError::WindowsSymlinkPermission`.
  Per-file symlink with a directory source walks the tree
  (`walk_files`, sorted for deterministic order) and links one file per
  mirrored target path; `symlink-dir` creates a single directory link with
  no walk. `Copy` copies bytes to each target; `copy-tree` collects the
  source tree (`collect_tree`, sorted) and mirrors each file. Template
  render reads the `.tmpl` body, calls `engine.render` **once**, then writes
  the same bytes to each target with the `.tmpl` suffix stripped
  (`strip_tmpl_suffix`, case-insensitive). Every executor returns one
  `CompletionRecord` per materialized object (per target, and per walked
  file for the directory cases) so T-010's progress cursor records one entry
  per object and backups/status/rollback inherit the granularity without
  special-casing multi-target.
