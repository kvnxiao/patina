- Completed: Landed `patina_core::apply::hooks` (REQ-006), the execution

  invocation-wide `--force-deploy` toggle, with a private `enforces`
  helper folding the override into the `must_succeed` decision);
  `HookError` (`ShellNotFound`, `When`, `Spawn`); `ResolvedHook<'_>` (a
  hook whose shell is resolved, borrowing the originating `HookEntry`);
  `HookOutcome` (`Succeeded` / `Warned` / `Failed`); and three public
  functions —
  - `resolve_shells(hooks, host_os)` resolves every hook's shell up front,
    defaulting an omitted `shell` to `bash` on macOS / Linux and `pwsh` on
    Windows and rejecting an explicit shell that does not resolve on PATH
    with `HookError::ShellNotFound` *before any hook runs*. The whole set
    fails if any one shell is unresolved (no partial runnable set).
  - `should_run(hook, engine, resolver)` returns `true` for a hook with no
    `when`, else evaluates the predicate through the shared T-008
    `MiniJinja` `Engine::eval_when`, mapping an undefined/eval failure to
    `HookError::When`.
  - `run_hook(hook, force_deploy)` (async) spawns the resolved shell
    (`-c` for POSIX shells, `-Command` for `pwsh`/`powershell`, selected on
    the shell's file stem) against the hook command via
    `tokio::process::Command`, awaits the exit status, and classifies:
    zero → `Succeeded`; non-zero → `Failed` when `must_succeed` is
    effectively enforced, else `Warned`. A non-zero *exit code* is a
    classification, not a `HookError`.

  PATH resolution is a private cross-platform helper (`resolve_on_path`)
  that checks a path-bearing shell directly and otherwise walks `PATH`; on
  Windows it fans out across `PATHEXT` so a bare `pwsh` resolves to
  `pwsh.exe`. The module returns *classifications* only — it never aborts,
  rolls back, or writes user-facing output; mapping `Failed`/`Warned` to
  the CLI exit codes (2 / 3 / 0) and firing rollback are the orchestrator's
  (T-016 / T-018) job, and the `HookEvent` on the borrowed entry tells the
  orchestrator which branch (`pre_apply` abort vs `post_apply` rollback).

  Wired into `patina-core/src/apply/mod.rs` (`pub mod hooks;` plus
  re-exports of the public surface), `patina-core/src/lib.rs` (root
