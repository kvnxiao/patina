At `HEAD` (3a5da46), neither the CLI command nor the core entry point
existed, so any test driving `patina watch --foreground` would fail to
parse (clap rejects the unknown subcommand) and `run_foreground` would
not resolve:

```

0     # no `Command::Watch` variant
$ git show HEAD:patina-core/src/watch/mod.rs | grep -c 'run_foreground'
0     # no foreground loop entry point
```

### Green — Scenario: SIGINT clean shutdown (CHK-008)

`foreground::sigint_shuts_down_cleanly_and_exits_zero` in
`patina-cli/tests/watch_foreground_cli.rs` applies a one-symlink module
(so a journal exists), spawns `patina watch --foreground` as a subprocess
with `RUST_LOG=patina_core=info` and piped stderr, waits until the loop
logs `watch_started`, sends `kill -INT`, and asserts the process exits
`0` within 1 second and that stderr contains `shutdown`.

```
$ cargo test -p patina-cli --test watch_foreground_cli
test foreground::sigint_shuts_down_cleanly_and_exits_zero ... ok
```

### Green — Scenario: `[watcher] debounce_ms` warning (REQ-006 `<done-when>` / DEC-002)

window. `no_watcher_table_does_not_warn` is the negative control (no
`[watcher]` table ⇒ no warning). Five `watcher_config_warning` lib unit
tests in `patina-core/src/watch/mod.rs` pin the pure helper (plain
manifest, `[watcher]` table without the key, the key present, and a
malformed manifest all classified correctly).

