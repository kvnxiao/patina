# Evidence for SPEC-0005 T-001

Red-then-green paper trail for T-001: introduce the `Disposition` type
and reset the on-disk format major from `2` to `1`. Backs the
`demonstrated` CHK roll-call entries in `journal/T-001.md`.

## Session 2026-06-02-T001-attempt-1

### Scenario 1 ā€” Envelope major byte reads as 1 (CHK-017)

Proves the on-disk major was actually reset, observed at the wire
byte rather than only at the constant.

<red>
Before the reset, `FILE_MAJOR_VERSION` was `2`
(`patina-core/src/journal/plan.rs:29`), so a freshly encoded `Plan` /
`ApplyRecord` carried `02 00` (little-endian `u16` 2) in its first two
bytes. A test asserting the leading major reads as `1` would have
failed:

```
assert_eq!(Plan::read_envelope_version(&bytes), 1);
// pre-reset: left == 2, right == 1  -> FAILED
```
</red>

<green>
After the reset to `1`, the new unit tests
`plan::tests::envelope_major_byte_reads_as_one` and
`record::tests::envelope_major_byte_reads_as_one` pass: the leading
`u16` is `01 00` and `read_envelope_version` returns `1`. Confirmed
under `cargo test --workspace --locked` (37 doctests + all unit/integration
tests green; `test result: ok` on every line).
</green>

### Scenario 2 ā€” A major-2 buffer is refused, not misdecoded (CHK-018)

Proves the supported-major lowering takes effect: a buffer prefixed
with major `2` is now rejected with `VersionMismatch { found: 2,
supported: 1 }` rather than decoded.

<red>
While `FILE_MAJOR_VERSION` was `2`, a buffer whose envelope major was
`2` equalled the supported major, so `decode_envelope` accepted it
(`found > supported` is false for `2 > 2`) and the body decoded
successfully. A test asserting that a major-2 buffer is *refused*
would have failed ā€” the decode returned `Ok(_)`, not
`Err(VersionMismatch)`.
</red>

<green>
After the reset, `decode_envelope` refuses `2 > 1`. The new tests
`plan::tests::major_two_buffer_is_refused_not_misdecoded` and
`record::tests::major_two_buffer_is_refused_not_misdecoded` overwrite
the leading `u16` with `2u16.to_le_bytes()` on an otherwise valid
encoded body and assert:

```
Err(JournalError::VersionMismatch { found: 2, supported: 1 })
```

Both pass under `cargo test --workspace --locked`.
</green>
