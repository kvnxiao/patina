the `Held`-policy re-journal (REQ-003, REQ-009, REQ-010). The red phase runs
the new `patina-cli/tests/remove_cli.rs` crate against a binary with the

### Scenario 1 — `patina remove ~/.zshrc --yes` replaces the symlink, drops the entry, and status omits the target (CHK-005)

`remove_replaces_target_drops_entry_and_status_omits_it`.

<red>
Pre-edit: the `Remove` subcommand wiring is stashed, so `patina remove
~/.zshrc --yes` is a clap usage error (exit 2). The exit-0 assertion fails;
the replace / entry-removal / status-omission assertions are never reached:

```
thread 'remove_replaces_target_drops_entry_and_status_omits_it' panicked at patina-cli/tests/remove_cli.rs:62:5:
assertion `left == right` failed: remove must exit 0; stderr: error: unrecognized subcommand 'remove'
  left: 2
 right: 0
```
</red>

<green>
Post-edit: the fixture first `patina apply --yes`s a `zsh` module whose
`[[file]]` symlinks `~/.zshrc` → `<repo>/zsh/zshrc` (content `shell-config`),
asserting `~/.zshrc` is a symlink. Then `patina remove ~/.zshrc --yes`:

- replaces `~/.zshrc` with a regular file (not a symlink) holding
  `shell-config` (the last-applied content reconstructed from the journaled
  source);
- removes the `[[file]]` entry from `<repo>/zsh/patina.toml` (no `[[file]]`
  remains);
- leaves the repository source `<repo>/zsh/zshrc` unchanged (still
  `shell-config`);
- a subsequent `patina status --json` lists no `files` entry whose basename
  is `.zshrc` (the fresh `<ts>.COMMIT` from the `Held`-policy re-apply omits
  it, so it is unmanaged/absent rather than ORPHANED).

All asserted; the test exits 0.

```
test remove_replaces_target_drops_entry_and_status_omits_it ... ok
```
</green>

`remove_unmanaged_path_exits_1`.

<red>

not the expected exit 1. The exit-1 assertion fails:

```
thread 'remove_unmanaged_path_exits_1' panicked at patina-cli/tests/remove_cli.rs:166:5:

  left: 2
 right: 1
```
</red>

<green>
Post-edit: against the applied-symlink fixture (so a commit record exists,
but for `~/.zshrc`, not `~/.bashrc`), `patina remove ~/.bashrc --yes` finds
no journaled `ExpectedTarget` whose `manage_key` matches `~/.bashrc`, so it:

- exits 1;
- writes a stderr message naming `~/.bashrc` and the phrase `not managed`
  (and the three discovery sources: `$PATINA_REPO`, walk-up, persisted
  default);
- leaves the unmanaged `~/.bashrc` file unchanged (`untouched`);
- leaves the managed `zsh` module's `[[file]]` entry intact (the refusal is
  before any mutation).

All asserted; the test exits 0.

```
test remove_unmanaged_path_exits_1 ... ok
```
</green>
